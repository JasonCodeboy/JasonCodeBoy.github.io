# 防抖 debounce

监听一个输入框的值，文字发生变化后触发 change 事件

直接用 keyup 事件，则会频繁触发 change 事件

防抖：用户输入结束或暂停时，才会触发 change 事件

const input1 = document.getElementById("input1");

let timer = null;

input1.addEventListener("keyup", function () {
  if (timer) {
    clearTimeout(timer);
  }
  timer = setTimeout(() => {
    console.log(input1.value); // 模拟触发change事件

    // 清空定时器
    timer = null;
  }, 500);
});

只有一个 input 就写了这么大一堆代码，太浪费，所以我们能不能封装一下呢

const input1 = document.getElementById("input1");

// 防抖
function debounce(fn, delay = 500) {
  // timer 是在闭包中的
  let timer = null;

  return function () {
    if (timer) {
      clearTimeout(timer);
    }
    timer = setTimeout(() => {
      fn.apply(this, arguments);
      timer = null;
    }, delay);
  };
}

input1.addEventListener(
  "keyup",
  debounce(() => {
    console.log(input1.value);
  }, 600)
);

  解析：debounce 函数执行的时候传入两个参数，第一个是函数，第二个是延迟时间，首先在 debounce 里面定义一个 timer，timer是在闭包中的（很重要），timer 是不对外暴露的，所以说我们调用函数的也是也是我们不需要关心的，因为 timer 不能被修改，如果被修改的话，函数里面的逻辑就会发生混乱，所以说别人也修改不了。然后 return 一个 function，function 要执行的话，如果 timer 有的话就清空，跟上面的代码逻辑一样，然后 timer 赋值成一个定时器，主要看定时器里面的 apply，fn.apply(this, arguments);这个是什么意思呢？就是说，函数在执行的时候，把当前的 this 以及 arguments 的参数传进来，因为返回的这个函数，执行的时候可能会传入一些参数，执行 debounce 的时候需要把参数传进来，apply 的 this 也需要把 this 传进来，当然如果你要使用 this 的话，debounce 函数调用的时候传入的函数就不能使用箭头函数了，只能使用 function 的形式，具体为什么不多说，前面已有讲解。最后就是 timer 赋值成 null 回到最初始。

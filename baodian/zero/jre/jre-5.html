# 节流 throttle

# 前言

节流保持一个频率连续触发

拖拽一个元素时，要随时拿到该元素被拖拽的位置

直接用 drag 事件，则会频繁触发，很容易导致卡顿

节流：无论拖拽速度有多快，都会间隔 100ms（时间自定） 触发一次

# 封装

// 节流函数封装
function throttle(fn, delay = 100) {
  let timer = null; // timer在闭包中

  return function () {
    if (timer) {
      return;
    }

    timer = setTimeout(() => {
      fn.apply(this, arguments);

      timer = null;
    }, delay);
  };
}

div1.addEventListener(
  "drag",
  throttle(function (e) {
    console.log(e.offsetX, e.offsetY);
  })
);

  apply：绑定 this 以及绑定参数。这个参数在什么地方有用呢？在下面调用的时候比如 e 这个参数，比如我们一般写一个div1.addEventListener("drag",function(e) {} )绑定事件，function 里面有一个 e 参数，这个参数是我们在绑定事件的时候它会触发事件的回调函数，然后把 e 这个事件对象传递进来，这是我们一般的使用习惯，但是我们用 throttle 去包裹一个函数的话，这个 e 会传递给谁？它会传递给 throttle 返回的函数，这个函数会接收 e，用 throttle 的方式来写，我们的 event 事件对象是传递给 throttle 返回的函数的，它不会直接传递给里面包裹的这个函数，那怎么让里面被包裹的函数有 e 这个事件对象呢，就需要通过 apply，arguments 传递进来才行，大家可以试一下 如果在定时器里面直接执行函数，不使用 apply，e 肯定会报错 xxx of undefined

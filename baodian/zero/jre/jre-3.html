# 性能优化

# 前言

  性能优化是一个综合性问题，没有标准答案，但要求尽量全面。你可以去谷歌百度搜索一下前端性能优化，很多答案，你把这些答案都拼在一块都可能能写成一本书，所以说这个问题特别考验综合性的能力

  某些细节问题可能会单独提问：手写防抖，节流。考察代码逻辑能力以及你是不是用过，另外还考察了应用编程的能力，因为这块编写的时候，很考验定时器的使用。这个并没有放到异步里面去写，因为异步只是一个实现的手段，所以它最根本的目的还是做一个性能优化。

只关注核心点，针对面试

# 性能优化原则

1. 多使用内存、缓存或其他方法

2. 减少 CPU 计算量、减少网络加载耗时

适用于所有编程的性能优化 —— 空间换时间

# 从何入手

1.让加载更快

2.让渲染更快

# 让加载更快

1. 减少资源体积：压缩代码

2. 减少访问次数：合并代码，SSR 服务器端渲染、缓存

3. 使用更快的网络：CDN

# 让渲染更快 - 1

1. CSS 放在 head，JS 放在 body 最下面

2. 尽早开始执行 JS，用 DOMContentLoadad 触发

3. 懒加载（图片懒加载，上滑加载更多）

# 让渲染更快 - 2

1. 对 DOM 查询进行缓存

2. 频繁 DOM 操作，合并到一起插入 DOM 结构

3. 节流 throttle、防抖 debounce

# 缓存

1. 静态资源加 hash 后缀，根据文件内容计算 hash

2. 文件内容不变，则 hash 不变，则 url 不变

3. url 和文件不变，则会自动触发 http 缓存机制，返回 304

# SSR

1. 服务器端渲染：将网页和数据一起加载，一起渲染

2. 非 SSR（前后端分离）：先加载网页，再加载数据，再渲染数据

3. 早先的 JSP ASP PHP ，现在的 Vue React SSR

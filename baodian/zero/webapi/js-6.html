# 存储

# 前言

存储比较简单一些，但它很常用，不算必考，但是也是应该掌握的知识点之一，虽然知识点比较少，不是很难。

# cookie

# 本身用于浏览器和 server 通讯

# 被 "借用" 到本地存储

  为什么说被借用来呢？因为localStorage和sessionStorage这两个是html5之后才提出来的，html5是在09年10年左右提出来的，那个时候我们的互联网已经发展的非常好了，移动互联网还没那么好，但是电脑已经很普及了，各种网站等等，那个时候刚刚发布html5，没有local和session，所以我们只能用cookie来做本地存储

# 可用document.cookie = "..." 来修改

cookie采用key value形式存储，同一个key会覆盖，不同的key会新增

cookie的价值不在于本地存储，而在于客户端和服务端进行通讯

# cookie的缺点

# 1. 存储大小，最大4kb

# 2. http请求时需要发送到服务端，增加请求数据量

# 3. 只能用document.cookie = "..." 来修改，太过简陋

# localStorage 和 sessionStorage

# 1. HTML5专门为存储而设计，最大可存5M

# 2. API简单易用 setItem， getItem

# 3. 不会随着 http 请求被发送出去

# 描述 cookie localStorage sessionStorage的区别

# 1. localStorage 数据会永久存储，除非代码或手动删除

# 2. sessionStorage 数据只存在当前会话，浏览器关闭则清空

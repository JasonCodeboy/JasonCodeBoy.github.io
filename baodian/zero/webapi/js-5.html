# 状态码

# xhr.readyState

0 表示没有初始化，还没有调用 send() 方法

1（载入）已调用 send() 方法，正在发送请求

2（载入完成）send() 方法执行完成，已经接收到全部响应内容

3（交互）正在解析响应内容

4（完成）响应内容解析完成，可以在客户端调用

# xhr.status

2xx - 表示成功处理请求，如 200

3xx - 需要重定向，浏览器直接跳转，如 301 302 304

  1. 301 永久重定向
  2. 302 临时重定向
  3. 304 资源未改变

4xx - 客户端请求错误，如 404 403

  1. 404 找不到
  2. 403 接口没有权限

5xx - 服务器端错误

# 跨域：同源策略，跨域解决方案

# 什么是跨域（同源策略）

ajax 请求时，浏览器要求当前页面和 server 必须同源（安全）

  什么是同源策略呢，举个例子：比如你做一个网站，叫做山寨淘宝，然后在网站里面会发送 ajax 请求，去请求淘宝的一些 api，你觉得可能吗？这肯定不可能对吧，为什么不可能呢，因为同源策略。你的网页和 server（服务端）必须是同源才可以，如果不同源浏览器就直接给你限制死了

同源：协议、域名、端口、三者必须一致

前端：http://a.com:8080/ ; server：https://b/com/api/xxx

  以上仅限浏览器，比如我们的搜索引擎，爬虫它不是浏览器发送的，它是server端发送的，搜索引擎的爬虫是服务端去做的请求去访问各个网站的一些网页，照样可以把数据拿到。想要攻击某个网站不同源的接口是可以发起攻击的，在server端可以发起攻击，因为server端并没有像浏览器那样有同源策略，所以可以发起攻击的，但是怎么去预防呢，那就是服务端后端的事情了。

# JSONP

1. 访问 https://baidu.com, 服务端一定返回一个html文件吗

服务端不一定返回一个html文件的内容，服务端可以拼接任何的内容返回，现在访问各种各样的动态网页，比如你要访问一个新闻或者邮箱等类型的网站，它都会根据用户的个人信息去帮你推荐一些你自己个性化的一些东西，返回给你 ，所以服务端可以任意拼接返回。

2. 服务器可以任意动态拼接数据返回，只要符合html格式要求

3. 同理于<script src="www.xxx.com/abc.js">

用Script去访问一个js地址的话就一定是返回js文件吗？不一定

script标签可以绕过跨域限制，服务器可以任意动态拼接数据返回，所以 script标签就可以获得跨域的数据，只要服务端愿意返回

<script>
  window.callback = function (data) {
  console.log(data);
};
</script>

<script src="http://127.0.0.1:8001/data/jsonp.js"></script>

# 手写一个简易的 ajax

function ajax(url) {
    const p = new Promise((resolve, reject) => {
        const xhr = new XMLHttpRequest()
        xhr.open("GET", url, true)
        xhr.onreadystatechange = function () {
            if (xhr.readyState === 4) {
                if (xhr.status === 200) {
                    resolve(JSON.parse(xhr.responseText))
                } else if (xhr.status === 404) {
                    reject(new Error("404 not found"))
                }
            }
        }
        xhr.send(null)
    })

    return p
}
const url = "/data/test.json"
ajax(url).then(res => console.log(res)).catch(err => console.error(err))

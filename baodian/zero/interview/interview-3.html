# 数组 slice 和 splice 的区别

1. 功能区别（slice - 切片，splice - 剪切）

2. 参数和返回值

3. 是否纯函数？

const arr = [10, 20, 30, 40, 50];

// slice 纯函数
const arr1 = arr.slice();
const arr2 = arr.slice(1, 4); //  [20, 30, 40]
const arr3 = arr.slice(2); // [30, 40, 50]
const arr4 = arr.slice(-2); // 截取最后两个 不管原数组多长

// splice 非纯函数（有副作用）
const spliceRes = arr.splice(1, 2, "a", "b", "c");
console.log(spliceRes, arr); // spliceRes: [20, 30]; arr: [10, "a", "b", "c", 40, 50]

# [10,20,30].map(parseInt) 返回结果是什么

1. map 的参数和返回值

2. parseInt 参数和返回值

const res = [10, 20, 30].map(parseInt);
console.log(res); // [10, NaN, NaN]
1
2

以上是简写，拆开应该怎么写呢

// 拆解
[10, 20, 30].map((num, index) => {
  return parseInt(num, index);
});
1
2
3
4

  对这个题目失望回答错误的同学呢，应该都是你自己比较眼高手低，没有进一步的做拆解，首先大家看明白 上面的写法和下面的写法是完全一样的，如果这一点你看不明白那是另外的问题了，如果你能看明白这一点，这个问题你就能答得对。

  分析：如果 num 是 10 的话，index 是 0，然后我们 parseInt 一个 10，一个 0，返回值是 10，parseInt 后面的第二个参数你写 10 和写 0 是一样的，它都是转换成 10 进制来返回，但是你如果再进行第二个：parseInt（20，1），这个时候 1 不符合 parseInt 转换的格式，所以返回的就不是一个数字，再往后 30，index 就是 2、那同理 2 也不符合 parseInt 的转换规则。具体关于 parseInt 的第二个参数具体是什么意思可自行百度查一下，总之这道题考察不是 parseInt 的第二个参数什么意思，考察的是 map 这个函数是什么意思。

# ajax 请求 get 和 post 的区别是什么

1. get 请求一般用于查询操作，post 一般用户提交操作

2. get 参数拼接在 url 上，post 放在请求体内（数据体积可能更大）

3. 安全性：post 易于防止 CSRF

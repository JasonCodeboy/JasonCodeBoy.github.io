# new 的实现

还是那句话，想实现什么，就得先了解它的原理。

如果对 new 的原理还不太理解的，可以看下个人之前的文章：原型。

进入正题：

首先让我们来简单回顾下 new 的原理：

1，在函数的最顶端 var this = { __proto__ : Fn.prototype }
    或者是：var this = Object.create(Fn.prototype)

2，逐步执行函数内部的代码

3，在函数的最底部 return this



因为 new 是操作符，所以我们需要用函数的形式来进行模拟实现，如下：

function Fn(a, b) {
    this.a = a
    this.b = b
}
Fn.prototype.c = '3'

var obj = new Fn('1', '2')

我们将用 var obj = myNew(Fn, '1', '2') 来实现上面的操作。

那么我们就可以开始着手实现了：

function myNew() {

    // 在函数的最顶端 var this = { }
    var _this = { }

    // 获取第一个参数，构造器
    // 注意，这里的 arguments 已经被改变了
    // 改变前 [Fn, '1', '2'] --> 改变后 ['1', '2']
    Fn = Array.prototype.shift.call(arguments)

    // 将实例的原型指向构造器的原型
    _this.__proto__ = Fn.prototype

    // 改变 this 后，逐步执行构造函数内部的代码
    Fn.apply(_this, arguments)

    // 在函数的最底部 return this
    return _this
}

那么试试我们自己的 myNew 吧：

var obj = myNew(Fn, '1', '2')
console.log( obj.a )    // '1'
console.log( obj.b )    // '2'
console.log( obj.c )    // '3'

到这里 new 的基本实现已经完成了。

但是我们还知道，如果在构造函数内部定义的返回值是对象类型或者函数类型，那么 new 基本就失效了，返回的就是自定义的对象或函数。如果是其他类型，那么无影响。

function myNew() {

    var _this = { }

    Fn = Array.prototype.shift.call(arguments)

    _this.__proto__ = Fn.prototype

    // 将构造函数的返回值赋值给 val
    var val = Fn.apply(_this, arguments)

    // 判断构造函数的返回值：
    // 如果是个对象，或函数，则返回 val
    // 其他类型，则返回 _this
    let res = typeof val === 'object' || typeof val === 'function'

    // val || _this 的意思是，如果构造器的返回值是 return null，则继续返回 _this
    return res ? val || _this : _this
}

以上，就是 new 的实现，知道了 new 的原理，实现起来就非常简单了。

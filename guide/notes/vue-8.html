    # 全局和局部注册组件，命名规范，props 作用和原理

    如何定义一个组件？当我们在 new Vue() 的时候，它就是一个组件。这是一种函数式的写法。

    const component = {
    template: '<div>这是一个组件</div>'
}

component 这时候仅仅是一个对象，它还不是 Vue 的组件，但是我们可以通过几种方式，去把它变成 Vue 的组件

    # 全局注册组件：

    首先，我们可以在全局使用 Vue 这个类的 component 方法，去把这个组件定义到全局去使用。

Vue.component('CompOne', component)

这样的话，我们就定义了一个叫 CompOne 的组件。

我们在页面所有的地方去使用 CompOne 写在 template 里面，它就代表我们使用了 component 组件

    为什么上面 2 个地方的命名方式不一样呢？

    定义组件名的时候，我们最好使用大驼峰式命名。 （因为定义的组件名是 JS 里面的变量，所以需要用驼峰式。而一个 Vue 的组件，我们可以认为它是一个类，那么类的命名，肯定是需要用大驼峰式）

    当写到 template 里面时，用全小写 + 连接符的方式。（因为 HTML 不区分大小写）

    这样的命名方式比较清晰。

    如果我们需要这个组件只在某个地方使用呢？

    那么我们需要新增一个配置项 components，然后按照 key : value 的形式去定义组件。

    key 就是组件的名字，value 就是组件的对象。 需要注意的是：

    我们在使用 data 定义组件内数据的时候，如果我们这一个组件不是通过 new Vue() 的方式去生成出来的，而是我们注册全局或者局部去使用的话，那么 data 必须要是一个 function，而且 return 的是一个新建的对象。

    那么为什么要这么去做呢？我们可以利用一个变量来逃避这种方法，如下图：

    我们发现，在上面一个组件输入，下面输入框的数字也跟着一起变。

    我们使用 2 个组件，肯定是希望它们的数据都是单独一份不一样的。

    而不是我们改了第一个组件的数据，结果第二个组件里面的数据也跟着一起改。

    props 是用来定义这个组件，在被外部使用的时候，它的一些可变的行为。

    也就是说，我们去定义一个组件的时候，我们希望的是这个组件帮我们解决一系列通用性的问题。

    就是说这个组件会指定一个功能，然后把这些功能包含进去，主要的目的就是处理这些功能。

    我们处理一些功能的时候，肯定不是每个地方都不是一模一样，它会需要一些不同的配置项，来配置这个组件的一些行为。

    虽然大方向上是解决同一个问题，但是在一些小的细节上会有一些差异，那么这些差异通过什么去体现？就是通过 props

    注意：
props 内的命名和组件的命名一样。

props: {
    // 大驼峰式命名
    propOne: String
}

在使用 propOne 的时候用连接符的形式。（Vue 会自动的去做处理，匹配到真正 props 内的名称）
<comp-one prop-one="xxx"></comp-one>

    注意：传进来的 props 是不能被修改的。

    因为我们在组件内部主动的去更改 props，其实是不对的，如果你要去主动改一个值，那么你这个值更应该定义在 data 里。

    props 是外部组件，用来约束这个组件的展示行为。

    并且，作为一个子节点，我们不应该去主动的修改父节点给我们的一些定义。

    如果我们要去修改父节点的一些定义，说明我们本身这个组件的行为和定义就没做好。

    那么如果我们要改 props，那么要怎么做呢？我们可以去触发一些事件，告诉外部的组件。

    # 方法一：

    虽然可能有点绕，但是这个就是 props 的单项数据流概念。因为如果你经常两边去改 props，它涉及到你的父组件和子组件。

    如果你子组件可以随便改父组件的内容，就会导致一个逻辑的混乱，开发中就会出现很多的问题。

    # 方法二：

    this.$emit 就是实例上面的 $emit。我们在子组件上 @change，实际上就是绑定了 $on。

    comp-one 其实就是一个 Vue 的实例。

    props 绑定事件的原理也就是：通过 $on 在 Vue 的实例上绑定一个事件，在通过 $emit 触发。

    # props 的数据验证：

    props: {
    active: {
        type: Boolean,
        required: true     // 必传，不然报 warn
    },
    propOne: {
        type: String,
        default: '123'     // 默认值
    },
    propTwo: {
        type: Array,
        default() {        // 如果默认值是个引用类型：数组
            return []
        }
    },
    propThree: {
        type: Object,
        default: () => {}  // 如果默认值是个引用类型：对象
    },
    propFour: {
        type: String,
        validator( 外部传入的val ){   // 更严格的验证
            // 在这里判断是否符合你的要求
            return typeof val === 'string'
        }
    },
}

# Event Loop（事件循环）机制的简单理解

这里主要是讲解，关于浏览器环境下 JS 引擎的事件循环机制。

关于 Event Loop，网上随便一搜都有很多专业的文章，写的非常详细和全面。

但是对于不懂的小伙伴们，文字太过于专业。反而在刚开始，不利于理解和构建脑海中的初步模型。

这里主要是个人对异步总结的一点小技巧，希望对初学者刚开始的理解，和做面试题有所帮助。

正题开始：

其实关于事件循环，个人总结就三句话：

先画三个圈圈，或者盒子。

# 1，同步代码，丢到 1 号 盒子。
# 2，promise.then() 中的代码丢到 2 号 盒子。（微任务：promise.then 里的回调函数会放到相应宏任务的微任务队列里，等宏任务里面的同步代码执行完再执行）
# 3，setTimeout 和 setInterval 中的代码丢到 3 号 盒子。（宏任务：settimeout / setInterval 的回调函数放到宏任务队列里，等到执行栈清空以后执行）

然后 1，2，3 号盒子中的代码顺序执行。

如下图： 需要注意的三点是：

1，Promise 中的异步体现在 then 和 catch 中，所以写在 Promise 中的代码是被当做同步任务立即执行的。

2，在 async / await 中，在出现 await 之前的代码，也是被当作同步立即执行的。

3，await 右边的代码是同步，await 下面的代码是异步。

关于 Event Loop（事件循环）机制，希望个人的总结能对刚接触这个概念的小伙伴们有所帮助。

最后，在附上一道题目，如果能做对，基本就已经理解了这个概念，再看关于异步的专业文章，也就不会在一头雾水了。

console.log("1");

async function async1() {
  console.log("2");
  await async2();
  console.log("3");
}

async function async2() {
  console.log("4");
}

setTimeout(function () {
  console.log("5");
  new Promise(function (resolve) {
    console.log("6");
    resolve();
  }).then(function () {
    console.log("7");
  });
});

async1();

new Promise(function (resolve) {
  console.log("8");
  resolve();
}).then(function () {
  console.log("9");
});

console.log("10");

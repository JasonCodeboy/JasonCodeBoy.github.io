    # bind 的实现

    要实现一个 bind，仍然得先了解这个 API 的作用：MDN 对 bind 的讲解和示例

    实现 bind 首先要理解偏函数的大致概念，MDN 上对偏函数的解释和示例过于专业。
刚接触的小伙伴可能看不太懂，这里简单铺垫下：

简单来说，偏函数就是：固定部分参数，生成另外一个参数更少的方法。

首先看一个最简单的 add 函数。
function add(a, b) {
    return a + b
}
如果是这样的求和函数：
add(1, 2)
add(1, 3)
add(1, 4)
add(1, 5)
......
由于第一个参数每次都是1，那么对于相同的参数，我们是不是可以想个办法，只需要传一次就OK呢？
那么就可以运用闭包的功能，比如：
function oneAdd (fn , a) {
    return function (b) {
        return fn(a, b)
    }
}
var res = oneAdd(add, 1)
res(2)
res(3)
res(4)
res(5)

    # bind 实现的第一步：

    首先一个简单的小例子：
this.x = '1'

var obj = {
    x: '2',
    getX: function () { return this.x }
}

var getX = obj.getX
getX() // 1

var getXBind = getX.bind(obj)
getXBind() // 2



可以看到，bind 和 call / apply 略有不同，bind 返回的是一个指定了 this 的函数。
那么基于这点，我们可以实现一个简单的 bind：

Function.prototype.myBind = function (obj) {
    // obj 代表指定的 this

    var fn = this // this 代表 bind 前面的函数

    return function () {
        return fn.apply(obj) // 等同于将 obj.fn() 执行后的返回值在 return 出去
    }
}

那么试试我们自己的 myBind：
var getXMyBind = getX.myBind(obj)
getXMyBind() // 2

    # bind 实现的第二步：

    然而，bind 也是可以传参的，并且和 call 一样，数量不固定，还要将传入的参数保留到下次运用。
同时下一次的传参也是不固定且有效的，那么怎么处理呢？偏函数在这里就用上了：

Function.prototype.myBind = function (obj) {
    var fn = this

    // 这里的作用是指，去除传入的第一个参数 this 的指向。保留剩下的参数到 args
    // 例如： fn.bind(this, 1, 2, 3, 4) --> args = [1, 2, 3, 4]
    var args = Array.prototype.slice.call(arguments, 1)

    return function () {

        // 将第二次传入的参数，存入 twoArgs
        // 由于 arguments 是类数组，所以需要转换成数组
        var twoArgs = Array.prototype.slice.call(arguments)

        // args.concat(twoArgs) 意思是合并2次传入的参数
        // 例如：fn.bind(obj, 1)(2, 3) --> fn.bind(obj, 1, 2, 3)
        // 等同于 obj.fn(1, 2, 3)
        return fn.apply( obj, args.concat(twoArgs) )
    }
}

function fn(a, b, c) {
    console.log( this.name )
    console.log( a )
    console.log( b )
    console.log( c )
}

var obj = {
    name: 'obj'
}

fn.bind(obj, 1)(2, 3)    // obj 1 2 3
fn.bind(obj, 1, 2)(3)    // obj 1 2 3

fn.myBind(obj, 1)(2, 3)  // obj 1 2 3
fn.myBind(obj, 1, 2)(3)  // obj 1 2 3

    # bind 实现的第三步：

    如果不考虑作为构造函数使用的绑定函数，那么上面一步就已经完成了。但是根据 MDN 上面的描述：

    # 绑定函数自动适应于使用 new 操作符去构造一个由目标函数创建的新实例。当一个绑定函数是用来构建一个值的，原来提供的 this 就会被忽略。不过提供的参数列表仍然会插入到构造函数调用时的参数列表之前。

    首先让我们看个小例子，理解下 MDN 上这句话的意思：

    function Fn (a, b) {
    this.a = a
    this.b = b
}
Fn.prototype.c = '3'

var fn = new Fn('1', '2')
fn.a              // '1'
fn.b              // '2'
fn.c              // '3'
fn instanceof Fn  // true


var obj = { }
var bind_Fn = Fn.bind(obj, '1')

var bind_obj = new bind_Fn('2')

// 注意：bind 时指定的 this ( obj ) 失效了，但是传入的参数依然有效

console.log( obj )  // { } --> obj 还是个空对象

bind_obj.a  // '1'
bind_obj.b  // '2'
bind_obj.c  // '3'

bind_obj instanceof Fn               // true
bind_obj instanceof bind_Fn          // true
new Fn('1', '2') instanceof bind_Fn  // true


这里简单描述下 instanceof 作用：

A instanceof B 的意思是：看 对象A 的原型链上 有没有 函数B 的原型

    那么接下来就轮到我们来完善最后一步了：

    Function.prototype.myBind = function (obj) {
  var fn = this;
  var args = Array.prototype.slice.call(arguments, 1);

  var F = function () {
    var twoArgs = Array.prototype.slice.call(arguments);

    // 如果被当作为构造函数使用时，F函数内部的 this 的指向就是实例
    // 那么这里就不能在直接写 obj 了 而是需要改成 this instanceof F ? this : obj

    // 这行代码的意思是：

    // this instanceof F 的值为 true：
    // 就代表着用了 new 操作符，则忽略传入的 this（obj），直接用其系统默认的 this（new的原理）

    // this instanceof F 的值为 false：
    // 则代表被当作普通函数执行，那么这时候的 this 肯定是指向 window，因此就将绑定函数的 this 指向 obj

    return fn.apply(this instanceof F ? this : obj, args.concat(twoArgs));
  };

  if (this.prototype) {
    // 请不要在意这里的规范，这样写是为了让刚接触的小伙伴们更好的理解
    // 这里的 this 指的是被当做构造函数使用的 fn (函数最上面的 fn)

    function f() {}
    f.prototype = this.prototype; // 声明一个新的 f 函数，来共享绑定函数的 prototype

    // 这里的作用就是修改 F函数的prototype 为绑定函数的 prototype
    // 其作用是让 new 出来的实例能够继承绑定函数上原型的属性或方法
    // 等同于 F.prototype = Object.create(this.prototype)
    F.prototype = new f();
  }

  return F;
};

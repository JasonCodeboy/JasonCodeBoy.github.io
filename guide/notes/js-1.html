    # 声明提前原理（预编译）

    我们在刚开始接触 JS 的时候，听到最多的就是声明提前，对于声明提前的解释，相信很多小伙伴们都牢记着一句话：把 var 和 function 声明的变量提升到代码块的最顶端就 OK。

    那么问题来了，以下这段代码执行后的结果是什么：

    console.log(a);
function a(a) {
  console.log(a);
  a = "1";
  console.log(a);
  var a;
  console.log(a);
  var a = function a() {
    console.log(a);
  };
  console.log(a);
  var a = "2";
  console.log(a);
  function a() {}
  console.log(a);
}
a("3");
var a = "4";
console.log(a);

    # 如果只是知道声明提前这句话，相信大家看到这道题就会有点摸不到头绪，那么接下来就让我们一起来揭开 JS 引擎执行规则的神秘面纱。（关于 JS 引擎的执行规则，网上和书上是有更底层和精准的讲解，下面的讲解只是为了让初学者更好的理解，主要作用于脑海中模型初次的构建，以及面对前端面试题）

    # 当我们在 script 标签中写完一段代码后执行，JS 引擎会走三个步骤：

    # 1，通篇扫描

    # 2，预编译

    # 3，解释一行，执行一行

    这里着重讲解的就是预编译。预编译简单来说分为四个步骤： 1，创建一个叫 AO 的空对象。（如果是全局则是 GO，函数内部则是 AO。为了让初学者更好的理解，以下都简称 AO。） 2，找所有用 var 声明的变量，和形参。并将其作为 AO 的属性名，同时赋值为 undefined。 3，将实参的值赋值给形参。 4，找函数声明，将其作为 AO 属性名，同时赋值为该函数体。

    例1：
console.log(a)
var a = 1;
console.log(a)
function a() { }
console.log(a)

// 1，创建空对象 AO
AO = { }


// 2，找所有用 var 声明的变量，和形参。并将其作为 AO 的属性名，同时赋值为 undefined
AO = {
    a: undefined
}


// 3，将实参的值赋值给形参
这里是全局，不是函数内部，所以跳过

// 4，找函数声明，将其作为 AO 属性名，同时赋值为该函数体。
AO = {
    a: function a(){}
}
这里简单讲解一下：
关于对象的键名，只能是字符串和 Symbol 类型。其他类型的键名会被隐式的转换成字符串类型，也就是会默认的调用 toString 方法。
那么这里就会发生覆盖，简单来说，就像 var obj = { num: 1 }，再执行 obj.num = 2 的操作一样。

通过预编译之后，所有 var 和 function 声明都会去掉，代码如下：

// AO = {
//   a: function a(){}
// }

console.log(a) // ƒ a(){ }
a = 1;
console.log(a) // 1
console.log(a) // 1

相信到了这里，答案是多少，应该一目了然了。

那么回到正题：
console.log(a)
function a(a) {
    console.log(a)
    a = '1'
    console.log(a)
    var a
    console.log(a)
    var a = function a() {
        console.log(a)
    }
    console.log(a)
    var a = '2'
    console.log(a)
    function a() { }
    console.log(a)
}
a('3')
var a = '4'
console.log(a)

全局的预编译环节：
// 1，创建空对象 AO
AO = { }

// 2，找所有用 var 声明的变量，和形参。并将其作为 AO 的属性名，同时赋值为 undefined
AO = {
    a: undefined
}

// 3，将实参的值赋值给形参
这里是全局，不是函数内部，所以跳过

// 4，找函数声明，将其作为 AO 属性名，同时赋值为该函数体。
AO = {
    a: function a(){}
}

经过预编译后，代码如下：
console.log(a) // ƒ a(){ }
a('3')
a = '4'
console.log(a)


然后开始执行，那么第一个 a 的打印，就是 ƒ a(){ } 了，随后函数 a()，这里又触发了一次预编译：
函数 a 内部的预编译环节：
function a(a) {
    console.log(a)
    a = '1'
    console.log(a)
    var a
    console.log(a)
    var a = function a() {
        console.log(a)
    }
    console.log(a)
    var a = '2'
    console.log(a)
    function a() { }
    console.log(a)
}
a('3')

// 1，创建空对象 a.AO（为了方便区分）
a.AO = { }

// 2，找所有用 var 声明的变量，和形参。并将其作为 AO 的属性名，同时赋值为 undefined
a.AO = {
    a: undefined （var 声明的变量 和 形参变量 都是 a，所以覆盖）
}

// 3，将实参的值赋值给形参
a.AO = {
    a: ‘3’
}

// 4，找函数声明，将其作为 AO 属性名，同时赋值为该函数体。
a.AO = {
    a: function a(){}
}

经过预编译之后的代码：
function a(a) {
    console.log(a) // ƒ a(){ }
    a = '1'
    console.log(a) // 1
    console.log(a) // 1
    a = function a() {
        console.log(a)
    }
    console.log(a) // ƒ a(){ }
    a = '2'
    console.log(a) // 2
    console.log(a) // 2
}

相信做到这步，答案也是一目了然了

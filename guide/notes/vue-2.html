<item>
    <div></div>
</item>
那么这个 div 就是作为这个 item 的 children，传入到这个 item 里面的。
我们在 item 里面调用 $children，就是这个 div。

    # app.$refs

    那么这个 $refs 是什么呢？它是我们模版里面的一个引用。

    我们可以看到下图，刚开始是一个空的对象。

    如果我们给 div 上面 加个属性 ref = 'div'，再打印出来，就是 div 对应的 html 节点

    那么这个 ref 有什么用呢？

    它可以帮我们去快速的定位到，我们模版里面的某一个节点、或者某一个组件。

    如果它是 html 节点，它会返回 html 节点的这个对象。

    如果它是一个组件，那么它会返回组件的实例。组件的实例也就是 Vue 的实例，我们可以通过像上面的操作方式一样，去操作 Vue 的实例，可以去修改一些属性或者其他的操作。

    # app.$isServer

    $isServer 这个属性，一般情况我们都不会用到。只有在我们服务端渲染的时候，我们才会去做这个判断。

    那么为什么会有它的存在呢？因为有一些代码，我们是可以在服务端渲染去运行，而有一些代码，我们是只能在客户端去运行。

    那么有些时候，我们的项目里面加入了服务端渲染，我们就需要有一个值来帮我们去判断，什么情况到哪里去运行。

    所以 Vue 就提供了这个值，我们每个 Vue 的实例都可以去通过这个属性进行判断。

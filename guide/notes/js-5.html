    # 各种情况下的 this 指向

    关于 this 的指向，不多说，就四句话：

    # 1，函数名()，this 指向 window。（函数预编译过程和全局作用域里，this 指向 window）

    function fn() { ... }
fn() // this --> window

直接用函数名执行底层是：test() 等同于 test.call() 执行

call 没传参数，或者传的 null 或 undefined， 运行环境就是 window

    # 2，obj.fn() ---> fn 里面的 this 指向 obj。

    var obj = {
    name: '对象',
    fn: function () { ... }
}
obj.fn() // this --> obj

谁通过 . 调用的 fn。 fn 内的 this 就指向谁



var arr = [fn, 1, 2, 3]
arr[0]() // this --> arr

函数作为数组的一个元素，通过数组下标调用的：this 指向这个数组




var name = 'A'
var obj = {
    name: 'B',
    prop: {
        name: 'C',
        getName: function () {
            return this.name;
        }
    }
}
obj.prop.getName() // getName 前面的 . 是 prop    this --> prop

var test = obj.prop.getName
test() // 无人调用，底层则是 test.call()    this --> window

    # 3，通过 new 调用， this 指向 new 出来的对象。

    function Test(name, age){
    // var this = { }

    this.name = name
    this.age = age

    // return this
}
var obj = new Test() // this --> obj


new 的作用：
1，在该函数内部的最顶端，隐式的加上 var this = { }。
2，依次执行函数内部代码。
3，最后，在函数内部的最下方，隐式的 return this

    # 4，通过 call / apply / bind 调用，this 指向 我们传参的参数对象

    var num = 123
Object.prototype.toString.call(num) // this --> num

// fn.call(obj) 等同于 obj.fn    this --> obj

简单来说，call 的作用就是：
让 num 去调用 call 前面的方法 Object.prototype.toString


fn.call( obj, 参数1, 参数2, 参数3, ... )       // this --> obj
fn.allply( obj, [ 参数1, 参数2, 参数3, ... ] ) // this --> obj
fn.bind( obj )( 参数1, 参数2, 参数3, ... )     // this --> obj
